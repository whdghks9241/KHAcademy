Getter 와 Setter
	객체지항 프로그래밍(OOP)의 중요한 개념 중 하나
	클래스의 맴버 변수에 접근하고 값을 설정하는 메서드
	메서드를 사용해서 객체의 상태를 보호하고, 객체간의 데이터를 안전하게 전달
	
Getter
	객체의 필드값을 setter와 vaild검증을 통한 후 반환하는 역할
	필드의 값을 가져와서 외부에서 읽게 해줌
	메서드의 이름은 get으로 시작하고 필드의 이름과 유사하게 사용한다
	
Setter
	객체의 필드 값을 설정 하거나 변경하는 역할
	필드 값을 변경하려면 Setter 메서드를 호출하여 값을 설정
	메서드의 이름은 set으로 시작하고 필드에 이름과 유사한 이름을 사용한다.
	Setter는 일반적으로 매개변수를 받아 필드의 값을 설정한다
	
	
Vaild
	Setter에서 넘어온 값이 무결성인지 검증하는 역ㅎ랄

캡슐화 : 필드를 private으로 선언하여 외부에서 직접 접근하지 못하도록 막아주면서
		Getter와 Setter를 통해 필드에 접근할 수 있도록 해줌
		
데이터의 유효성 검사 : Setter 메서드를 사용하여 데이터를 설정하기 전에 유효성을 검사하고 필요한 검사 논리를 추가할 수 있음.

코드 유지 보수 : 프로그램이 계속 발전하고 변경될 때, Getter와 Setter를 사용하면 필드와 관련된 로직을 집중적으로 관리할 수 있음
